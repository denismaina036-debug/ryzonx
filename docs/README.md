# RyvonX Documentation

## Purpose

This directory is the permanent source of truth for RyvonX platform documentation. It centralizes architecture specifications, engineering implementation specifications, design references, and supporting reference materials used across all phases of development.

## Structure

| Directory | Purpose |
|-----------|---------|
| `architecture/` | Platform architecture and business specifications |
| `implementation/` | Engineering implementation specifications organized by development phase |
| `design/` | UI/UX references and visual assets |
| `references/` | Supporting documentation and reference material |

## Legacy Documents

The following files predate the current documentation system and are retained for reference only:

- [ARCHITECTURE.md](./ARCHITECTURE.md) — early foundation architecture notes
- [ROUTING.md](./ROUTING.md) — early routing map
- [NAMING_CONVENTIONS.md](./NAMING_CONVENTIONS.md) — code naming quick reference

For authoritative specifications, use `architecture/` and `implementation/` instead.

## Usage

- [12_PLATFORM_REFINEMENT.md](./architecture/12_PLATFORM_REFINEMENT.md) — product simplification sprint (workflows & messaging)
- Implementation specifications guide **engineering** work and reference architecture as source of truth.
- Design documents define **how** the platform should look and behave in the UI.
- Reference documents provide shared vocabulary and approved visual references.
